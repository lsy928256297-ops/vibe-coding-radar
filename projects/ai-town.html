Case Position

Vibe Coding 案例定位

AI 小镇适合作为“好玩互动”路线的 Vibe Coding 项目案例：新手能快速理解目标，也容易用截图、录屏或公开 demo 展示成果。

为什么适合做案例

角色会移动、聊天、留下记忆，结果很可视化。它能让搜索 “Vibe Coding 案例” 的用户看到 AI 编程不只是表单和后台。

案例最小版本

3 个角色、1 个场景、10 条记忆、每天一段小剧情。先证明互动体验，再考虑地图、经济系统或多人协作。

MVP

先做这个效果

01场景

一张小地图

只保留一个可走动区域，不要第一版就做复杂城镇、任务系统和多场景切换。

02角色

三个 NPC

每个 NPC 有一句人设、一个目标和几条记忆，能互相对话就足够构成演示。

03剧情

每天生成一段小剧情

先做自动生成日志，而不是复杂任务编排。用户能看到角色“活着”，项目就有 wow moment。

Blockers

可能会卡在哪里

  1. 后端和实时状态

    AI Town 这类项目通常不只是静态页面，会涉及状态同步、数据库或实时后端。新手要先确认本地开发命令和服务依赖。

  2. LLM API key

    如果示例依赖模型 API，需要先确认环境变量、额度和调用失败时的降级方案。

  3. 范围膨胀

    不要同时加地图编辑器、复杂经济系统和多人联机。第一版只证明“角色能行动、能对话、能留下记忆”。
