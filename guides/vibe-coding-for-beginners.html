Vibe Coding 新手怎么开始

如果你第一次用 Codex、Cursor 或 Claude Code 做项目，不要先问“我要做一个完整 App 吗”。 更好的起点是：选一个能快速看到反馈的项目，让 AI 先做项目体检，再跑最短 demo。

看项目推荐 看 AI 小镇例子 看必装 Skill
建议周期 2h-2d

先把一个小切片跑通，再决定要不要加登录、数据库、部署和长期维护。

Start Here

三步开工法

这个流程适合大多数 Vibe Coding 项目，也适合把 GitHub 项目交给 AI 做体检。

01选题

先选有即时反馈的项目

小游戏、白板、海报、PDF 工具、周报机器人和 ESP32 传感器都适合新手，因为你能很快判断“它动起来了吗”。

02体检

让 AI 先读项目再动手

先让 AI 读 README、安装文档、示例和依赖文件，列出 API key、Docker、数据库、硬件和网络卡点。

03MVP

只跑最短可演示切片

不要第一天做完整产品。先让它完成一个能截图、能分享、能讲清楚的最小效果。

FAQ

新手最常问的三个问题

  1. Vibe Coding 新手第一步应该做什么？

    选一个 2 小时到 1 天能跑出效果的项目，复制原始 GitHub 或参考链接，让 AI 先判断最短 demo、环境要求和可能卡点。

  2. 哪些项目不适合第一天做？

    复杂登录权限、支付、重后端、多租户、私有平台 API、大模型训练和重硬件焊接，都不适合作为第一天项目。

  3. AI 写坏了怎么办？

    不要急着重写。让 AI 先复现错误、读日志、定位改动范围，再做最小修复。能跑测试或浏览器验收时，先验收再继续加功能。
