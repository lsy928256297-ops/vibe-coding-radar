AI 编程工具

新手选 AI 编程工具，不要追求最多，而要追求闭环：能读项目、能改代码、能跑验证、能部署公开链接。

看 Codex 怎么用 零基础 AI 编程 用 AI 做项目
原则 闭环

工具组合要覆盖读取、修改、验证、部署，而不是只覆盖聊天和补全。

Stack

一套够用的 AI 编程工具栈

Agent主力

Codex / Claude Code

负责在项目目录里读文件、改代码、跑命令和做验证，适合推进从想法到 demo 的完整过程。

Editor编辑

Cursor / Copilot

适合在编辑器里解释代码、局部修改、补全函数和理解文件。它们更像手边的代码副驾。

Deploy验收

GitHub / Vercel / Playwright

GitHub 管来源，Vercel 管公开链接，Playwright 或浏览器检查负责证明页面真的能用。

Prompt

复制给 Codex 的工具选择提示词

请你根据我的项目目标，帮我选择一套最少但够用的 AI 编程工具。请按读取代码、编辑代码、运行验证、部署公开链接、错误排查五个环节列出推荐工具，并说明哪些工具现在不需要。目标是减少切换成本，先把一个最小 demo 跑通。
这页面向搜索 “AI 编程工具” 的用户，负责解释工具分工，并把工具选择导向可执行项目。
