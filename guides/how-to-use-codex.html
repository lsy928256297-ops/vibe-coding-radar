Codex 怎么用

Codex 最适合用来做项目推进：读代码、改文件、运行命令、分析错误、做验证。新手要给它明确目标和边界。

用法 先读后改

让 Codex 先理解项目，再小步执行，并用证据验收。

Workflow

Codex 做项目的标准流程

01读取

先让它读上下文

让 Codex 读取 README、package、脚本、入口文件和相关文档，先总结项目结构和风险。

02执行

一次只做一个改动

让 Codex 先说要改哪些文件，再执行最小修改。不要同时让它改 UI、后端、部署和数据结构。

03验证

用命令或浏览器验收

跑语法检查、测试、构建、curl 或浏览器截图。没有验证结果，就不要说完成。

Prompt

复制给 Codex 的标准开工提示词

请你作为 Codex 项目搭档，先读取当前项目的 README、依赖文件、入口文件和相关文档，判断这个项目的结构、运行方式、可能卡点和最短 demo 路径。请先给出执行计划，说明要改哪些文件、如何验证、哪些事情暂时不做。执行时小步推进，遇到错误先诊断日志，不要直接重写整个项目。
