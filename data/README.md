## 数据范围

数据集分为 `fun`、`useful`、`hardware` 三条路线，每条 30 项。每条记录包括中文项目名、简介、技术栈、建议的最小可行版本、好玩度、实用度、上手度、原始来源、演示地址和可用的雷达案例页。

`source_url` 是项目维护方仓库或项目主页；`radar_case_url` 只在雷达已经完成来源、最小验收和局限核验时提供。评分和 MVP 是 Vibe Coding 雷达的编辑建议，不代表上游项目维护方意见。

## 局限

外部仓库的维护状态、许可证、安装方式和演示地址可能变化。数据集不替代开工前的仓库核验，也不把推荐分数当作通用基准。涉及 API、付费额度、Docker、数据库或硬件型号时，应再次查阅对应项目的一手文档。
