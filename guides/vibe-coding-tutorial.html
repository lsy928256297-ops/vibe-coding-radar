Vibe Coding 教程

这是一条适合新手的 Vibe Coding 教程：不要先追求完整产品，先用一个能看到结果的小项目，练习提示词、项目体检、调试和部署。

看 AI 新手编程 用 AI 做项目 Codex 怎么用
目标 小步可验收

一次只推进一个可验证结果：能打开、能点击、能生成、能部署。

Steps

五步完成一个 Vibe Coding 小项目

01想法

把愿望写成最小效果

把“我想做一个 App”缩成“第一屏输入什么，点按钮后输出什么”。目标越小，AI 越不容易跑偏。

02体检

让 AI 先读再改

如果有仓库或模板，先让 AI 读 README、脚本、依赖和示例，判断安装成本、API key、数据库和部署风险。

03上线

拿到公开链接

能部署的项目更容易复盘。第一版不需要完美，但要能被别人打开，看到你做出的核心效果。

Prompt

复制给 Codex 的教程提示词

请你作为 Vibe Coding 教练，带我完成一个小项目。请先把我的想法缩成 2 小时内能看到结果的最小 demo，再做项目体检：需要哪些文件、命令、依赖、API key、数据库或部署准备。每一步只推进一个可验证结果，并告诉我如何验收。不要一开始就做完整产品。
这页面向搜索 “Vibe Coding 教程” 的用户，负责把教程型流量引导到项目实践和 Codex 开工提示词。
